# CSA Notes – Design Rationale

---

## [EN] Why This Structure Was Chosen

Most welfare systems fail due to:

- User dependency
- Administrative overload
- Delayed response

Mulberry avoids these by:

- Automating execution, not judgment
- Removing user interaction requirements
- Embedding finance as a passive rail

---

## [KR] 설계 선택의 이유

기존 복지 시스템의 실패 원인:

- 사용자 의존
- 행정 과부하
- 지연된 지원

Mulberry의 대응:

- 판단이 아닌 집행만 자동화
- 사용자 개입 요구 제거
- 금융을 ‘보이지 않는 레일’로 설계
