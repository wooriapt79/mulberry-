# Contributing to Mulberry AI Investment Platform

We love your input! We want to make contributing to this project as easy and transparent as possible, whether it's:

- Reporting a bug
- Discussing the current state of the code
- Submitting a fix
- Proposing new features
- Becoming a maintainer

## Development Process

We use GitHub to host code, to track issues and feature requests, as well as accept pull requests.

## Pull Requests

1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. If you've changed APIs, update the documentation.
4. Ensure the test suite passes.
5. Make sure your code lints.
6. Issue that pull request!

## Code Style

- **Python**: Follow PEP 8
- **JavaScript/TypeScript**: Follow Airbnb style guide
- Use meaningful variable names
- Add comments for complex logic
- Write docstrings for all functions

## Commit Messages

We follow the [Conventional Commits](https://www.conventionalcommits.org/) specification:

```
feat: add new skill transfer algorithm
fix: correct Spirit Score calculation
docs: update database design documentation
style: format code according to PEP 8
refactor: simplify NFT marketplace logic
test: add tests for collaborative learning
chore: update dependencies
```

## Testing

```bash
# Run all tests
pytest tests/

# Run specific test file
pytest tests/test_skill_system.py

# Run with coverage
pytest --cov=src tests/
```

## Documentation

- Update README.md if needed
- Add/update docstrings
- Update relevant documentation in `docs/`

## Bug Reports

Use GitHub Issues and include:

- Quick summary and/or background
- Steps to reproduce
- Expected behavior
- Actual behavior
- Notes (possible fix, environment info, etc.)

## Feature Requests

We love feature requests! Please include:

- Clear description of the feature
- Why it's needed
- How it should work
- Any implementation ideas
